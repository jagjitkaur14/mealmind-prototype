# MealMind Prototype — Smart Weekly Meal Planner

MealMind is an interactive product prototype for a smart weekly meal-planning experience. It helps busy working professionals plan meals based on their dietary preferences, pantry items, available cooking time, and weekly routine.

This project was created as a product-management focused prototype to demonstrate user problem definition, core user flows, feature prioritization, and a clickable front-end concept.

---

## Project Overview

Busy professionals often struggle with the daily question: **“What should I cook today?”** This leads to decision fatigue, repeated food choices, grocery waste, and last-minute unhealthy meals.

MealMind solves this by helping users:

- Generate a 7-day meal plan
- Use ingredients already available in their pantry
- Respect dietary preferences and dislikes
- Keep cooking time within a preferred limit
- Auto-create a shopping list for missing ingredients
- Swap meals when needed

---

## Target User

### Primary Persona: The Overwhelmed Professional

**Priya, 29**, is a working professional who cooks at home but has limited time after work. She wants simple, healthy meals but does not want to spend too much time planning, shopping, or deciding what to cook.

### User Characteristics

- Age: 25–40
- Working professional
- Time-poor
- Health-conscious
- Budget-aware
- Cooks at home but needs planning support

---

## Problem Statement

Busy individuals need a smart meal-planning tool that adapts to their available cooking time, dietary needs, and pantry inventory so they can eat well, reduce food waste, and avoid daily meal-planning stress.

---

## Key Problems Identified

| Problem | User Impact |
|---|---|
| Decision fatigue | Users waste time deciding what to cook every day |
| Pantry waste | Ingredients expire because users forget what they already have |
| Diet drift | Users fall back on unhealthy or repetitive quick meals |
| Shopping inefficiency | Users buy duplicate items or forget required ingredients |
| Generic meal plans | Existing tools do not fully consider pantry stock, time limits, and personal preferences |

---

## Solution

MealMind creates a personalized weekly meal plan by combining three important inputs:

1. **User preferences** — dietary restrictions, cuisine choices, dislikes, allergens, servings, and skill level
2. **Pantry inventory** — ingredients already available at home
3. **Cooking constraints** — maximum cooking time and meals per day

The product then generates a weekly plan and automatically creates a shopping list for missing ingredients.

## User Flows

### 1. First-Time Setup

```text
Sign Up → Set Preferences → Add Pantry Items → Generate Plan → Review & Swap → Shopping List
```

### 2. Weekly Replanning

```text
Open Dashboard → Update Pantry → Regenerate Plan → Adjust Swaps → Export List
```

### 3. Mid-Week Meal Swap

```text
Click Meal Card → View Alternatives → Swap Meal → Shopping List Auto-Updates
```

## Prototype Screens

The prototype includes the following product screens:

1. **Dashboard** — weekly calendar, meal stats, and quick actions
2. **Generate Plan** — guided weekly meal generation flow
3. **Pantry Manager** — add, remove, and categorize pantry items
4. **Shopping List** — checklist for missing ingredients
5. **Preferences** — dietary restrictions, cooking time, cuisine, allergens, and disliked ingredients
6. **Meal Modal** — meal details, ingredients, instructions, and swap option

---

## Product Decisions

| Decision | Reason |
|---|---|
| Pantry-first planning | Reduces grocery waste and improves relevance |
| 30-minute cooking goal | Matches the needs of busy working users |
| Meal swap option | Gives users flexibility without restarting the plan |
| Shopping list automation | Reduces manual planning effort |
| Preferences-first setup | Makes recommendations more personalized |

---

## Success Metrics

If MealMind were launched as a real product, these metrics would help measure success:

| Metric | Target |
|---|---|
| Task completion rate | More than 85% of users can generate a meal plan successfully |
| Time to generate plan | Less than 2 minutes |
| Pantry match rate | More than 70% of planned meals use existing ingredients |
| Shopping list usage | Users check off or export the list weekly |
| Repeat usage | Users return weekly to regenerate or update plans |
| User satisfaction | Positive feedback after first week of use |

## Usability Test Plan

### Test Tasks

1. Set dietary preference to vegetarian and max cooking time to 30 minutes
2. Add 5 pantry items
3. Generate a weekly meal plan
4. Swap Tuesday dinner
5. Export or use the shopping list

### What to Observe

- Can users understand the dashboard without explanation?
- Do users know where to add pantry items?
- Is the meal generation flow clear?
- Can users easily swap a meal?
- Is the shopping list useful and easy to act on?
